# Astra USB Installer / USB Backup Manager

Автоматический мониторинг USB-устройств, создание резервных копий файлов и управление устройствами через GUI.

## Возможности

- Автоопределение новых USB-устройств (Linux / Windows)
- Монтирование, сканирование и инкрементальное копирование файлов
- Одна папка на устройство: `USB_Backups/Device1/`, докачка только новых/изменённых файлов
- Запись `.astra_id` на USB для постоянной идентификации устройства
- GUI на Tkinter (fullscreen, вкладки: поиск, устройства, workers, настройки)
- Выход из GUI только по паролю (смена пароля через вкладку настроек)
- Headless-режим (Docker, сервер без экрана)
- Поддержка нескольких USB одновременно (ThreadPoolExecutor)
- SQLite-база: история устройств и бэкапов
- Поиск по дате, устройству, человеку

## Быстрый старт (Docker)

```bash
docker compose up -d --build
```

Для работы с GUI на хосте нужен X-сервер. Docker пробрасывает `/tmp/.X11-unix`.

## Использование без Docker

```bash
pip install -r requirements.txt
python main.py          # автоопределение: GUI или headless
python usb_monitor.py   # принудительно headless
```

## Пароль на выход

Приложение работает в полноэкранном режиме. Для выхода требуется пароль.

- При первом запуске пароль: `exit`
- Пароль можно сменить во вкладке **Настройки** → **Изменить пароль**
- Хранится в `data/config.json` (файл в `.gitignore`, не попадает в репозиторий)
- Переменная окружения `APP_EXIT_PASSWORD` задаёт пароль при первом запуске

## Инкрементальное копирование

Каждое устройство получает одну папку: `USB_Backups/Device{id}/`.

- При повторном подключении файлы сверяются по размеру и дате изменения
- Неизменённые файлы пропускаются
- Новые файлы копируются
- Изменённые файлы сохраняются с суффиксом даты (старый не удаляется)
- `.astra_id` не копируется

## Голос нанокостюма при запуске

При старте GUI проигрывается приветствие голосом нанокостюма. Источник голоса
выбирается автоматически по убыванию качества:

1. **Точная копия (клонирование голоса, XTTS v2)** — даёт ТОЧНО тот же голос,
   что в игре. Требует один короткий референс-клип реального голоса
   нанокостюма:
   - Возьмите 6–15 секунд чистой речи нанокостюма из русской локализации
     (один-два WAV/MP3, без музыки и шума) и сохраните как
     **`data/nanosuit_ref.wav`** (моно/стерео, любой sample rate).
   - Путь можно переопределить переменной `NANOSUIT_REF_WAV`.
   - Установите движок: `pip install -r requirements-voice.txt`
     (кроссплатформенно: Arch Linux и Windows). При первом запуске скачивается
     модель (~1.8 ГБ).
   - XTTS воспроизведёт фразу «С возвращением, Пророк» этим же голосом. Так
     как референс уже содержит «костюмный» тембр, дополнительные DSP-эффекты
     не накладываются.
2. **Нейросетевой голос Silero** — если референс-клипа нет: живой русский
   мужской голос + «нанокостюмный» эффект. Ставится тем же
   `pip install -r requirements-voice.txt` (модель ~60 МБ скачивается один раз
   в `data/`).
3. **espeak-ng + DSP** — если не установлены опциональные голосовые зависимости.

Все зависимости голоса опциональны: без них приложение работает, просто
приветствие беднее по качеству.

> Лицензия весов XTTS v2 (CPML) — некоммерческая; для личного использования
> подходит.

### Диагностика голоса

Если голос **одинаковый с референс-клипом и без него** — значит клонирование
не запускается и играет запасной движок (Silero). Проверьте, какой движок
реально используется и почему остальные пропущены:

```bash
python gui.py --voice-test
```

Команда печатает выбранный движок и точную причину пропуска клонирования
(например, `coqui-tts unavailable (ModuleNotFoundError: No module named 'TTS')`
— значит `pip install -r requirements-voice.txt` не прошёл).

Полезные переменные окружения:

| Переменная | Значение | Эффект |
|---|---|---|
| `NANOSUIT_VOICE` | `clone` / `silero` / `espeak` / `sapi` | Принудительно один движок — ошибки видны, без тихого отката |
| `NANOSUIT_CLONE_FX` | `1` | Наложить металлический эффект костюма поверх клона (по умолчанию выкл.) |
| `NANOSUIT_REF_WAV` | путь | Свой путь к референс-клипу |

Например, `NANOSUIT_VOICE=clone python gui.py --voice-test` заставит запускаться
только клонирование и покажет реальную ошибку, если оно падает.

## Переменные окружения

| Переменная | По умолчанию | Описание |
|---|---|---|
| `USB_BACKUP_DEST` | `./USB_Backups` | Куда сохранять бэкапы |
| `NANOSUIT_REF_WAV` | `./data/nanosuit_ref.wav` | Референс-клип для клонирования голоса |
| `USB_DB_PATH` | `./data/devices.db` | Путь к SQLite БД |
| `USB_MAX_WORKERS` | `10` | Потоков для параллельной работы |
| `USB_DEBUG` | `0` | Включить отладку (1) |
| `DISPLAY` | `:0` | X11 display для GUI |

## Структура проекта

- `usb_monitor.py` — ядро: мониторинг, монтирование, копирование, SQLite
- `gui.py` — графический интерфейс (Tkinter)
- `main.py` — точка входа (GUI → headless fallback)
- `start.sh` — entrypoint для Docker
- `docker-compose.yml` — продакшн-развёртывание
- `Dockerfile` — сборка образа
- `requirements.txt` — зависимости Python
